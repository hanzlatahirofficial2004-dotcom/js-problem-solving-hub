Welcome to my dedicated space for mastering **JavaScript Fundamentals**! This repository is a curated collection of logic-building exercises, algorithmic challenges, and daily code snippets designed to sharpen problem-solving skills and write clean, optimized code.

## 🚀 Key Highlights
* **Zero Built-in Shortcuts:** Many challenges are solved without relying on high-level array methods (like `.reverse()`, `Math.max()`, or `.includes()`) to master underlying algorithmic logic.
* **Clean & Modular Code:** Focused on readability, performance, and best practices.
* **Structured Learning:** Divided cleanly into daily challenges and practical word problems.

## 📂 Repository Structure & Solved Challenges

### 📝 Week 1: Core Fundamentals

#### 🔹 Word Problems (`week_01_Word_Problems.html`)
Real-world scenarios converted into logical JavaScript functions:
* **Highest Scorer Finder:** Parses an array of student objects `[{name, score}]` to extract the top performer dynamically.
* **Candy Distributor:** A smart mathematical function calculating even distribution and remaining leftovers.
* **CRM Data Cleanliness:** Analyzes contacts database to calculate the exact percentage of non-duplicate, clean records.

#### 🔹 Daily Challenges (`week_01_Daily_Challanges.html`)
Algorithmic puzzles focused on strings, arrays, and pure logic:
* **String Reversal:** Reversing strings manually from scratch without `.reverse()`.
* **Missing Number Detector:** Finding the broken link in sequential arrays like `[1, 2, 4, 5]`.
* **Palindrome Validator:** Custom text analyzer checking symmetry without built-in methods.
* **Extreme Values Finder:** Locating the absolute minimum and maximum values without `Math.min()` or `Math.max()`.
* **Duplicate Finder:** Scanning arrays to isolate and extract repeated elements.
* **Array Intersection:** Finding common ground between two datasets without stacking `.filter()` and `.includes()`.
